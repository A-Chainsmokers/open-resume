# 测试指南

## 项目概述

在线简历编辑工具。前端 Vue 3 独立运行，后端 FastAPI 提供 AI 和 PDF 导出接口。简历数据存储在浏览器 localStorage，不依赖数据库。

## 测试范围

### 1. 简历编辑器

| 测试项 | 操作 | 预期 |
|---|---|---|
| 基本信息编辑 | 修改姓名、求职意向、手机、邮箱等字段 | 输入实时更新到预览区 |
| 富文本编辑 | 在个人优势、工作概述中使用工具栏 | 支持加粗、斜体、列表等格式 |
| 教育经历增删 | 新增一条教育经历，填写学校/专业/时间，删除 | 条目可新增、可删除 |
| 工作经历增删 | 新增工作经历，勾选"至今" | 结束时间隐藏，预览显示"至今" |
| 项目经历增删 | 新增项目，填写描述、技术栈、职责、成果 | 技术栈支持逗号分隔输入 |
| 技能增删 | 新增技能分类，添加多个技能项并选择熟练度 | 分类和技能项可增删 |
| 模块排序 | 在设置弹窗中调整"模块顺序" | 编辑器面板和预览区顺序同步变化 |
| 条目排序 | 点击条目的上移/下移按钮 | 条目顺序调整 |
| 深色主题 | 设置弹窗切换主题 | 整体 UI 切换为深色 |
| 自动保存 | 编辑后等待 600ms | 刷新页面后编辑内容不丢失 |

### 2. 模板预览

| 测试项 | 操作 | 预期 |
|---|---|---|
| 切换模板 | 设置弹窗切换 Classic/Modern/Minimal | 预览区模板样式切换 |
| 预览模式 | 点击顶部"预览" | 隐藏编辑器，全屏显示简历 |
| 编辑模式 | 点击顶部"编辑" | 恢复编辑器 |
| 响应式 | 缩小浏览器窗口宽度 | 移动端显示调整，侧边栏可折叠 |

### 2b. 自定义模板

| 测试项 | 操作 | 预期 |
|---|---|---|
| 导入合法模板 | 点击"导入模板 JSON"，选择一个合法的模板 JSON 文件 | 提示导入成功，模板出现在模板选择下拉框中 |
| 导入非法 JSON | 选择格式不正确的 JSON 文件 | 提示"格式不正确" |
| 自定义 CSS 生效 | 导入一个包含 CSS 样式的模板 JSON，切换为该模板 | 预览区应用了自定义 CSS |
| 删除自定义模板 | 点击自定义模板列表中的"删除" | 模板从列表和选择器中移除 |
| 持久化 | 导入一个模板后刷新页面 | 自定义模板仍在选择器中 |
| AI 生成提示词 | 展开"AI 生成提示词"折叠区域 | 显示可复制的提示词模板 |

### 3. 历史版本

| 测试项 | 操作 | 预期 |
|---|---|---|
| 保存版本 | 点击左侧"保存版本" | 弹窗提示保存成功 |
| 查看历史 | 点击左侧"历史版本" | 弹窗列出所有历史记录 |
| 差异展示 | 打开历史弹窗 | 每条历史记录下方展示与当前编辑内容的差异字段 |
| 字段变更显示 | 修改一个字段后保存版本，再修改该字段 | 差异中显示旧值（红底删划线）→ 新值（绿底） |
| 回退 | 点击"回退" → 确认 | 简历恢复为该版本内容 |
| 单条删除 | 点击"删除" → 确认 | 该记录消失，其余不变 |
| 清空全部 | 点击"清空全部" → 确认 | 所有历史记录消失 |
| 版本上限 | 连续保存 21 次 | 只有最新的 20 条，最早的被移除 |

### 4. PDF 导出

| 测试项 | 操作 | 预期 |
|---|---|---|
| 导出 | 设置弹窗点击"导出 PDF" | 浏览器下载一份 PDF 文件 |
| 内容完整性 | 打开导出的 PDF | 包含所有填写的信息，排版与预览一致 |
| 模板一致性 | 分别用三套模板导出 | 各模板样式正确 |

### 5. JSON 导入导出

| 测试项 | 操作 | 预期 |
|---|---|---|
| 导出 JSON | 设置弹窗点击"导出 JSON" | 下载 `.json` 文件 |
| 导入 JSON | 选择导出的 JSON 文件 | 编辑器填充为文件中的简历内容 |
| 导入异常 | 导入一个不合法的 JSON 文件 | 前端给出错误提示 |

### 6. AI 功能

需要先配置 AI（设置弹窗 → AI 配置 → 填写接口地址、API Key、模型）。

#### 润色

| 测试项 | 操作 | 预期 |
|---|---|---|
| 润色 | 在富文本字段上方点击"AI 润色" | 字段内容被改写，按钮显示加载状态 |
| 空内容 | 字段为空时点击润色 | 提示"请先输入需要润色的内容" |
| 配置缺失 | 清空 API Key 后点击润色 | 后端返回 400 错误提示 |

#### 聊天修改

| 测试项 | 操作 | 预期 |
|---|---|---|
| 聊天修改 | 输入自然语言指令（如"把工作经历改成三年"） | 简历内容按指令更新 |
| @引用 | 输入 @ 选择工作或项目条目 | 指令针对指定条目生效 |

#### PDF/图片识别

| 测试项 | 操作 | 预期 |
|---|---|---|
| PDF 识别 | 上传一份 PDF 简历 | 弹出确认框 → 确认后自动填充编辑器 |
| 图片识别 | 上传简历截图 | 同 PDF 识别，支持 .png/.jpg |
| 文件类型校验 | 上传非图片/非 PDF 文件 | 提示"请上传图片文件"或"请上传 PDF 文件" |
| 大小限制 | 上传超过 8MB 图片或 10MB PDF | 提示文件超出大小限制 |

### 7. AI 配置

| 测试项 | 操作 | 预期 |
|---|---|---|
| 保存配置 | 填写接口地址、API Key、模型后保存 | 配置持久化，下次打开自动填入 |
| 模型列表 | 在 AI 配置弹窗点击"获取模型" | 从配置的接口地址拉取模型列表 |
| 错误处理 | 填写错误的 API Key 后获取模型 | 提示"获取模型失败/401" |

### 8. 后端接口

| 接口 | 验证点 |
|---|---|
| `GET /api/health` | 返回 `{"status": "ok"}` |
| `POST /api/pdf/export` | 传入完整简历 JSON，返回 PDF（Content-Type: application/pdf） |
| `POST /api/ai/models` | 传入 `{baseUrl, apiKey}`，返回模型列表或错误 |
| `POST /api/ai/polish` | 传入 `{field, html, context}`，返回润色后的 html |
| `POST /api/ai/chat-edit` | 传入 `{message, resume}`，返回修改后的 resume 片段 |
| `POST /api/ai/recognize-pdf` | multipart 上传 PDF 文件，返回结构化简历数据 |
| `POST /api/ai/recognize-image` | 传入 `{image, mimeType}`，返回结构化简历数据 |

`api/test_main.http` 文件中包含 PDF 导出接口的请求示例，可在 VS Code 中安装 REST Client 扩展后直接发送测试。

### 9. 边界与异常

| 测试项 | 操作 | 预期 |
|---|---|---|
| 空简历 | 清空所有字段后导出 PDF | 生成空白 PDF，不崩溃 |
| 超长文本 | 在富文本字段输入大量文字 | 正常编辑、保存、导出 |
| 后端未启动 | 前端直接访问 | AI 接口返回 502/代理错误 |
| 并发操作 | 快速点击保存版本、回退、删除 | 不出现数据错乱 |
| 跨页面 | 新开一个标签页访问 | 数据来自同一 localStorage，保持一致 |
