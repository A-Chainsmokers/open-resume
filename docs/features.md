# 功能说明

## 简历编辑器

### 基本信息

| 字段 | 说明 |
|---|---|
| 姓名 | 文本输入 |
| 求职意向 | 文本输入 |
| 性别 | 文本输入 |
| 年龄 | 文本输入，如"26岁" |
| 手机号 | 文本输入 |
| 邮箱 | 文本输入 |
| 所在城市 | 文本输入 |
| 工作年限 | 文本输入，如"5年工作经验" |
| 期望薪资 | 文本输入，如"13-14K" |
| 期望城市 | 文本输入 |
| 头像图片地址 | 图片 URL，预览区会渲染为圆形头像 |
| 个人网站 | 文本输入 |
| GitHub | 文本输入 |
| 个人优势 | 富文本编辑器，支持加粗、斜体、列表等 |

### 教育经历

每条教育经历包含：

- 学校
- 专业
- 学历
- 开始时间 / 结束时间（年月选择器）
- 描述（多行文本列表）

支持新增、删除、上移、下移排序。

### 工作经历

每条工作经历包含：

- 公司
- 职位
- 城市
- 开始时间 / 结束时间（年月选择器）
- 至今（复选框，勾选后结束时间隐藏，预览显示"至今"）
- 概述（富文本编辑器）

支持新增、删除、上移、下移排序。

### 项目经历

每条项目经历包含：

- 项目名称
- 角色
- 开始时间 / 结束时间（年月选择器）
- 项目链接
- 项目描述（纯文本）
- 技术栈（逗号分隔输入，自动转为标签）
- 职责（富文本编辑器）
- 成果（富文本编辑器）

支持新增（插入到首位）、删除、上移、下移排序。

### 技能

按分类组织，每个分类包含：

- 分类名称（如"前端"、"后端"）
- 技能项列表，每项包含：
  - 技能名称
  - 熟练度（不显示 / 了解 / 熟悉 / 熟练 / 精通）

支持新增分类、添加技能、删除技能、上移、下移排序。

### 模块排序

左侧导航栏列出所有模块，在设置弹窗中可通过上移/下移调整各模块在编辑器中的排列顺序，预览区同步变化。

## 模板

支持三套内置模板和自定义模板导入，切换后预览区即时更新：

| 模板 | 类型 | 布局风格 |
|---|---|---|
| Classic | 内置 | 传统中文简历布局，顶部联系方式 + 头像，正文分模块排列 |
| Modern | 内置 | 双栏布局，侧栏为联系方式与技能，主栏为工作与项目经历，模块标题使用英文 |
| Minimal | 内置 | 紧凑极简风格，精简头部信息，正文分栏排列 |
| 自定义 | JSON 导入 | 由用户定义，可覆盖 CSS、布局、标签、渲染模式等 |

### 自定义模板

设置弹窗 → 自定义模板 → 导入模板 JSON，支持：

- **导入**：上传符合 `TemplateDef` 格式的 JSON 文件
- **删除**：删除已导入的自定义模板
- **CSS 注入**：模板 JSON 的 `css` 字段会作为 `<style>` 注入预览页
- **AI 生成**：提供提示词模板，可让 AI 生成模板 JSON 后直接导入

#### 模板定义字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 唯一标识 |
| `label` | string | 显示名称 |
| `cssClass` | string | CSS 类名 |
| `css` | string | 自定义 CSS 样式（可选） |
| `layout` | `center` / `sidebar` / `compact` | 整体布局 |
| `sidebarSections` | string[] | 侧边栏显示的区块 |
| `labels` | object | 各章节标题 |
| `currentText` | string | 当前职位日期文字 |
| `profileOutput` | `rich` / `text` | 个人优势输出方式 |
| `workOutput` | `rich` / `highlights` | 工作经历输出方式 |
| `projectsOutput` | `rich` / `list` / `compact` | 项目经历输出方式 |
| `educationOutput` | `detailed` / `simple` | 教育经历输出方式 |
| `skillsOutput` | `with-level` / `flat` | 技能输出方式 |

## AI 功能

所有 AI 功能需要先配置接口地址、API Key 和模型。

配置入口：设置弹窗 → AI 配置。

### 润色

每个富文本编辑区域（个人优势、工作概述、项目职责、项目成果）上方都有"AI 润色"按钮。点击后将该字段内容发送到 AI 进行专业化改写，改写结果直接替换原内容。

### 聊天修改

侧边栏打开"AI聊天"，通过自然语言对话修改简历。支持 @ 符号引用具体的工作经历或项目条目，使 AI 指令更精准。

### PDF 识别

上传 PDF 简历文件（限制 10MB），后端提取文本后调用 AI 解析为结构化简历数据，自动填充编辑器。

### 图片识别

上传简历截图（限制 8MB，支持 .png/.jpg），调用 AI 视觉模型识别并填充编辑器。

## 历史版本

所有历史数据保存在浏览器 localStorage。

### 版本创建

触发保存版本的操作：

| 操作 | 标签 |
|---|---|
| 点击左侧"保存版本" | 手动保存 |
| 导入 JSON | 导入 JSON |
| AI 聊天修改应用 | AI 聊天修改 |
| 图片识别填充 | 图片识别填充 |
| PDF 识别填充 | PDF 识别填充 |
| 切换模板 | 切换模板 |
| 重置简历 | 重置简历 |

版本上限 20 条，超出时自动丢弃最旧版本。如果当前内容与最新版本相同，不会重复创建。

### 版本展示

打开历史版本弹窗后，每条历史记录下方直接展示该版本与当前编辑内容之间的差异。

差异按模块分组展示：

- 旧值：红色背景 + 删除线
- 新值：绿色背景
- 未变化的字段不显示

### 版本操作

| 操作 | 行为 |
|---|---|
| 回退 | 恢复为该版本内容，后续版本被截断 |
| 删除单条 | 仅移除该记录，其余历史不变 |
| 清空全部 | 移除所有历史记录 |

## PDF 导出

后端使用 Playwright（headless Chromium）将简历渲染为 A4 PDF。

导出触发方式：设置弹窗 → "导出 PDF"。

后端接收完整简历 JSON，按当前选中模板渲染 HTML 后转为 PDF，返回文件下载。

---

## JSON 导入导出

### 导出

设置弹窗 → "导出 JSON"，下载一个包含当前简历完整数据的 `.json` 文件。

### 导入

设置弹窗 → "导入 JSON"，选择一个之前导出的 `.json` 文件，编辑器填充为该文件内容。导入会自动保存当前编辑内容到历史版本。

## 本地存储

| Key | 内容 |
|---|---|---|
| `resume_editor_current_resume` | 当前简历完整数据，编辑后 600ms 自动保存 |
| `resume_editor_version_history` | 历史版本列表，最多 20 条 |
| `resume_editor_ai_settings` | AI 接口配置（baseUrl、apiKey、model） |
| `resume_editor_theme` | 主题偏好（light / dark） |
| `resume_custom_templates` | 自定义模板列表 |

## 后端接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/health` | 健康检查 |
| GET | `/api/debug/routes` | 查看已注册路由 |
| POST | `/api/pdf/export` | 导出 PDF，接收 resume + template_id |
| POST | `/api/ai/models` | 获取模型列表，接收 baseUrl + apiKey |
| POST | `/api/ai/polish` | AI 润色，接收 field + html + context |
| POST | `/api/ai/chat-edit` | AI 聊天修改，接收 message + resume |
| POST | `/api/ai/recognize-pdf` | 识别 PDF 简历，multipart 上传 file |
| POST | `/api/ai/recognize-image` | 识别图片简历，接收 base64 image |
